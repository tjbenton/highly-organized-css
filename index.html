BEM

Block | Element | Modifier

en.bem.info/methodology, getbem.com/introduction

Solves the problems of Specificity, Context, & Consistency

.block {}
.block__element {}
.block__element--modifier {}

What are BEM's Benefits?

  • Avoids Nesting (single-level specificity)
  • Clear picture of what CSS & HTML are doing
  • No Guessing
  • Clearly denotes components
  • Scales marvelously

What are it's drawbacks?

  • Longer classes

Two Schools of Thought on approaching nested classes

Both agree not to do this

This is terribly verbose and also forces a DOM structure upon us; it is very inflexible

.list
  .list__header
  .list__items
    .list__items__item
      .list__items__item__title
      .list__items__item__content
      .list__items__item__actions
  .list__footer

1. Flattening everything to be on the same level

.list
  .list__header
  .list__items
    .list__item
      .list__title
      .list__content
      .list__actions
  .list__footer

2. Separate those elements

.list
  .list__header
  .list__items
      .list-item
        .list-item__header
        .list-item__body
  .list__footer

A way to avoid this problem is to not Block your scope too high up.

    .room__laptop

// app/lib/styles/base/_defaults.styl
// Axiomatic CSS
* {
  // Makes border color inhert the border color of the 
  // parent element and not the color.
  border-color: inherit;
  // Resets the box-model to what it should have been.
  box-sizing: border-box;
  margin: 0; // Removes margin from all elements
  outline: none; // Removes `outline` from all elements

  &:after, &:before {
    // pseudo elements inherit it's parent box-sizing style.
    box-sizing: inherit;
    // Pseudo elements are absolutely positioned by default 
    // because it's more common than not
    position: absolute;
  }

  // This should reduce the bloat of declaring vertical spacing on every 
  // single element/component.
  // http://alistapart.com/article/axiomatic-css-and-lobotomized-owls
  &:not(script):not(link):not(meta):not(style) ~ * {
    margin-top: $spacing;
  }
}

// Resets the root elements to not have a margin.
body > * {
  margin-top: 0;
}

html {
  // Sets the default color, and font for the site
  color: $config.color; 
  font: $config.font-size/$config.line-height $config.font-family; 
  // Ensures elements that are off the page won't 
  // effect the current layout
  overflow-x: hidden;
  // Default iOS settings
  -webkit-tap-highlight-color: rgba(black, 0);
  -webkit-text-size-adjust: 100%;
  -webkit-touch-callout: none;
  -ms-touch-select: none; // Enable WP8 fastclick
}

is-/has- State

format: .[is|has]-state { }

  • A short-lived style temporary states of the UI that need styling accordingly.
  • Never write a bare State class.
  • Bound to specific classes.
<div class="c-modal is-open is-loading">
  ...
</div>

.c-modal {
  ...
  &.is-open { ... }

  &.is-loading { ... }
}

js- Javascript

format: .js-component-name[<element>] { }

Used to bind javascript to elements.

Why?

  • You shouldn't bind styling and behaviour, otherwise we can't have one without the other.
  • Allows you to start reusing your js like you do your css.
  • Allows you to opt in to behaviours.
  • Spend less time refactoring your js.
  • Makes your js more reusable
<ul class='c-list c-list--limit js-toggle'>
  <li class='c-list__item'> ... </li>
  <li class='c-list__item'> ... </li>
  ... 10 more
  <li class='c-list__item'>
    <a class='js-toggle__trigger' data-state="is-open" data-text='Show Less' href=''>Show More</a>
  </li>
</ul>

$(document).on('click', '.js-toggle__trigger', function(e) {
  e.preventDefault()
  var $obj = $(this)
  var $text_data = $obj.data('text')
  if ($text_data) {
    $obj.data('text', $obj.text()).text($text_data)
  }
  $obj.closest('.js-toggle').toggleClass($obj.data('state') || 'is-active')
})

Conditional Viewports

format: .[class]@<breakpoint>{}

Viewport suffixes help keep things consistent and our namespaces intact

.o-show { } /* all viewports */

@media (min-width: 768px) {
  .o-show\@md { } /* only the medium size viewport and up */
}

<div class="u-hide u-show@md"> ... </div>

Ordering The Namespaces

s|t|(c|o|u)|((c|o|u)@(sm|md|lg|xl))|qa|js|is|has
  1. scope
  2. theme
    1. page, component, object, utility
    2. [page, component, object, utility]@[sm, md, lg, xl]
  4. qa, js, is, has
